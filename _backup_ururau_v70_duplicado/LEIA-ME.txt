-- O QUE FAZER SE O PAINEL FECHAR SOZINHO -------------------

Em v63, isso nao deveria mais acontecer. Mas se acontecer:

1. Execute INICIAR.bat (a versao COM console visivel).
2. Leia a mensagem de erro que aparece antes do "Pressione qualquer tecla".
3. Abra: logs\painel_inicializacao.log para ver o stack completo.
4. Causas mais comuns:
   - Codigo 2: rode INSTALAR.bat (falta python-dotenv)
   - Codigo 3: rode ATUALIZAR_DEPS.bat
   - Codigo 5: ururau.db corrompido — apague e reinicie
   - Codigo 6: erro de tkinter — reinstale Python com Tcl/Tk

-- COMO INSTALAR (apenas na primeira vez) -------------------

1. Instale o Python 3.10 ou superior:
   https://www.python.org/downloads/
   IMPORTANTE: marque Add Python to PATH durante a instalacao!

2. Extraia este ZIP em: C:\Projetos\Claude\ururau_v11\

3. Clique duas vezes em:
   INSTALAR.bat
   (aguarde terminar - pode demorar 2 a 5 minutos)

4. Abra o arquivo .env com o Bloco de Notas e preencha:
   - OPENAI_API_KEY  -> sua chave da OpenAI
   - URURAU_LOGIN    -> seu login do CMS
   - URURAU_SENHA    -> sua senha do CMS


-- COMO ABRIR O PAINEL -------------------------------------

Clique duas vezes em:
   INICIAR.bat

O painel abrira em uma janela grafica escura.


-- COMO ABRIR O ROBO 24H -----------------------------------

Clique duas vezes em:
   MONITOR.bat            -> Modo padrao: salva como RASCUNHO
   MONITOR_PUBLICAR.bat   -> Modo avancado: publica DIRETAMENTE

ATENCAO: MONITOR_PUBLICAR.bat publica materias ao vivo sem
revisao humana. Use somente apos configurar bem os criterios
de score e confianca minima no .env.


-- ARQUIVOS PRINCIPAIS -------------------------------------

INSTALAR.bat          -> Instala tudo (apenas uma vez)
INICIAR.bat           -> Abre o painel editorial
MONITOR.bat           -> Robo 24h em modo rascunho
MONITOR_PUBLICAR.bat  -> Robo 24h em modo publicacao direta
ATUALIZAR_DEPS.bat    -> Atualiza as dependencias
.env                  -> Suas credenciais (PRIVADO - nao compartilhe)
.env.exemplo          -> Modelo do .env
fontes_rss.json       -> Lista de feeds RSS para coleta
ururau.db             -> Banco de dados (criado automaticamente)
imagens/              -> Fotos processadas
logs/                 -> Logs do sistema (monitor.log, painel.log)


-- DOIS FLUXOS EDITORIAIS ----------------------------------

FLUXO 1: ROBO 24H (MONITOR_PUBLICAR.bat)
  -> Publicacao DIRETA no CMS (nao salva como rascunho)
  -> Score minimo mais alto (padrao 48 — configuravel em .env)
  -> Confianca minima obrigatoria (padrao 70 — configuravel)
  -> Maximo 4 publicacoes por hora
  -> Maximo 4 pautas da mesma fonte por ciclo
  -> Log detalhado de cada decisao (aprovacao/rejeicao + motivo)

FLUXO 2: PAINEL EDITORIAL (INICIAR.bat)
  -> Salva como RASCUNHO no CMS (requer revisao humana)
  -> Score minimo menor (padrao 28)
  -> Humano revisa, edita e decide publicar

-- BOTOES DO PAINEL ----------------------------------------

  Atualizar  -> Recarrega a fila de pautas
  Coletar    -> Busca novas pautas nos feeds RSS
  Redigir    -> Gera a materia da pauta selecionada
  Copydesk   -> Revisao profissional com IA
  Preview    -> Editar materia + escolher imagem + modo rascunho/publicar
  Publicar   -> Envia ao CMS (padrao: rascunho)
  Imagem     -> Busca imagem automatica para a pauta
  Manual     -> Adiciona pauta manualmente
  Descartar  -> Descarta a pauta selecionada
  Console    -> Mostra/esconde log interno do sistema


-- FLUXO RECOMENDADO (PAINEL) -------------------------------

1. Clique em Coletar
2. Selecione uma pauta na lista (use setas do teclado)
3. Verifique a aba Checagem
4. Clique em Redigir (aguarde IA gerar a materia)
5. Revise as abas Materia e Risco
6. Clique em Preview para editar e escolher imagem
7. No Preview: escolha Rascunho (recomendado) ou Publicar!
8. Clique "Enviar ao CMS"
